# 临时文件管理 Skill

> 统一管理临时文件：禁止创建临时文档，允许临时脚本但需及时清理

## 触发条件

### 临时文档管理（禁止创建）
- 用户要求整理项目、总结操作、生成报告时
- AI 准备创建 `.md` 总结文件时
- AI 准备删除已创建的总结文件时

### 临时脚本管理（允许但需清理）
- 用户说"运行临时脚本"
- 用户说"测试脚本"
- 用户说"临时测试"
- 用户提到 `_` 开头的 Python 文件

## 第一部分：临时文档管理

### 禁止创建的文件类型
- `整理完成总结.md`
- `操作报告.md`
- `临时总结.md`
- `项目整理报告.md`
- 任何包含"总结"、"报告"、"临时"的 `.md` 文件

### 允许创建的文件类型
- 正式的项目文档（README.md、API文档等）
- 代码文件（.py、.js等）
- 配置文件（.json、.yaml等）
- 测试文件（test_*.py）

### 文件操作规范
1. **创建前检查**：
   - 检查文件是否已存在
   - 确认文件是否符合项目规范
   - 验证文件是否应该提交到 git

2. **创建时规范**：
   - 使用正确的文件命名规范
   - 添加必要的文件头注释
   - 确保内容完整且有价值

3. **删除前检查**：
   - 确认文件是否真的需要删除
   - 检查文件是否被其他文件引用
   - 验证删除是否会影响项目功能

### 替代方案
**当需要记录操作结果时：**
1. **直接在对话中总结**：不需要创建文件
2. **更新现有文档**：如 README.md、CHANGELOG.md
3. **使用 git commit message**：在提交信息中详细说明

**当需要整理项目时：**
1. **直接执行整理操作**：不需要创建总结文件
2. **更新项目文档**：将整理结果更新到现有文档中
3. **使用 git 历史**：通过 git log 查看操作历史

## 第二部分：临时脚本管理

### 识别临时脚本
临时脚本的特征：
- 文件名以 `_` 开头（如 `_check.py`, `_fix.py`, `_patch.py`）
- 位于项目根目录
- 不是正式的测试文件（不在 `tests/` 目录）

### 执行流程
1. **运行脚本**：
   ```bash
   python <script_name>.py
   ```

2. **记录输出**：
   - 保存脚本输出到 `private/logs/` 目录
   - 文件名格式：`<script_name>_<timestamp>.log`

3. **删除脚本**：
   ```bash
   rm <script_name>.py
   ```

4. **确认结果**：
   向用户报告：
   - 脚本执行状态（成功/失败）
   - 输出摘要
   - 脚本已删除确认

### 注意事项
- 如果脚本执行失败，**不要删除**，保留供调试
- 所有日志保存在 `private/logs/`（已在 .gitignore 中）
- 长期使用的脚本应移动到 `private/scripts/`

## 示例

### ❌ 错误做法 - 临时文档
```
AI：我已整理完项目，创建了"整理完成总结.md"文件
AI：等等，这个文件不应该提交，我删除它
```

### ✅ 正确做法 - 临时文档
```
AI：我已整理完项目，主要改动如下：
1. 重组了目录结构
2. 更新了 .gitignore
3. 优化了代码组织
这些改动已通过 git commit 记录。
```

### ❌ 错误做法 - 临时脚本
```
AI：我创建了 _check.py 来测试功能
AI：脚本运行成功，但我不删除它了
```

### ✅ 正确做法 - 临时脚本
```
AI：运行 _check_i18n.py
1. 执行 `python _check_i18n.py`
2. 保存输出到 `private/logs/check_i18n_20260617.log`
3. 删除 `_check_i18n.py`
4. 报告结果：脚本执行成功，检查了15个文件的国际化支持
```

## 验证规则

### 临时文档验证
- [ ] 文件名是否包含"总结"、"报告"、"临时"
- [ ] 文件是否真的需要创建
- [ ] 是否有更好的替代方案（对话总结、更新现有文档）
- [ ] 文件是否应该提交到 git

### 临时脚本验证
- [ ] 文件名是否以 `_` 开头
- [ ] 脚本执行是否成功
- [ ] 输出是否已保存到日志
- [ ] 脚本是否已删除（成功时）或保留（失败时）

## 相关 Skill
- `doc-gen`：文档生成（不包括临时总结文件）
- `code-gen`：代码生成（不包括临时脚本）
- `error-recovery`：错误恢复（不涉及文件创建/删除）

## 设计说明
本 skill 整合了原 `no-temp-files` 和 `temp-script` 两个 skill，统一管理项目中的临时文件。核心原则是：
1. **临时文档**：禁止创建，直接在对话中总结或更新现有文档
2. **临时脚本**：允许创建但必须及时清理，执行后删除，失败时保留调试
